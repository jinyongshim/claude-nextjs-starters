# 코드 리뷰

인자: `$ARGUMENTS` (선택 — 파일 경로. 없으면 git 변경 파일 기준)

## 지시사항

다음 항목을 순서대로 검토하고, 발견된 문제를 파일/라인 번호와 함께 보고한다.

### 검토 대상 파일 결정

- 인자가 있으면 해당 파일만 검토
- 인자가 없으면 `git diff --name-only HEAD` 로 변경된 파일 목록 확인 후 전체 검토

### 검토 항목

#### 1. TypeScript 타입 안전성
- `any` 타입 사용 여부 (금지)
- 타입 단언(`as`)의 남용 여부
- 함수 반환 타입 누락 여부
- 인터페이스/타입 중복 정의 여부

#### 2. 보안
- 미인증 접근 가능한 API/페이지 여부
- 사용자 입력값 검증(Zod) 누락 여부
- Supabase RLS(Row Level Security) 우회 가능성
- 민감 정보 하드코딩 여부 (API 키, 비밀번호 등)

#### 3. Next.js 16 / React 19 패턴
- 서버 컴포넌트에서 `createClient()` 올바른 사용 여부 (`lib/supabase/server.ts`)
- 클라이언트 컴포넌트에서 서버 전용 모듈 import 여부
- `cookies()`, `headers()` 등 비동기 API에 `await` 사용 여부
- 불필요한 `"use client"` 지시문 여부

#### 4. 성능
- 불필요한 리렌더링 유발 가능성 (useEffect 의존성 배열 등)
- N+1 쿼리 패턴 여부
- 큰 컴포넌트 분리 필요 여부
- `Image` 컴포넌트 미사용 (일반 `<img>` 태그 사용 여부)

#### 5. CLAUDE.md 규칙 준수
- 절대경로(`@/`) 미사용 여부
- 들여쓰기 2칸 준수 여부
- 컴포넌트 네이밍 PascalCase 여부
- 에러 핸들링 누락 여부
